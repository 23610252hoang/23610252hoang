## 強み

### 1. 技術と業務をつなげて考える

ビリヤード会計アプリでは、画面を作るだけでなく、店舗スタッフの操作、料金計算、前払い、割引、追加注文、領収書、日次売上という業務フロー全体を設計しました。

### 2. 数字をそのまま信じず、検証する

与信スコアリング研究では、高いAccuracyだけを成果とせず、データリーケージ、公平性、説明可能性、別データセットへの汎化性まで確認しました。

### 3. 再現性と説明性を意識する

各プロジェクトで、実行手順、技術選定、課題と解決、結果、改善案をREADMEにまとめています。第三者がコードを確認し、再実行できる状態を目指しています。

## 開発時に意識していること

- 最初に「誰の、どの作業を改善するのか」を言語化する
- ベースラインを作ってから改善効果を比較する
- 個人情報や機密データを公開リポジトリに含めない
- エラーや例外を想定し、確認可能な出力を残す
- 技術的な結果を、利用者・採用担当者にも理解できる言葉で説明する
